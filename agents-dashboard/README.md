# agents-dashboard

Угловой виджет для OBS: какие сессии Claude Code сейчас активны. Самодостаточный прототип — свой мини-сервер на `:8723`, основной оверлей (`:8722`) не трогает. Слияние в основной `server.mjs` — отдельный следующий шаг.

## Запуск

```bash
node agents-dashboard/server.mjs        # http://127.0.0.1:8723
```

## Подключение в OBS

1. Источник → **Browser**.
2. URL: `http://127.0.0.1:8723/`
3. Width: `380`, Height: `220` (карточка 360px, высота по числу сессий).
4. Включить **Shutdown source when not visible**.

Превью без OBS: открыть `http://127.0.0.1:8723/?demo` — тёмный фон + фейковые данные.

## Что показывает

Одна строка на живую сессию Claude Code:

- **точка** — пульсирует зелёным (#4ade80), когда агент «думает»;
- **проект** — последний сегмент cwd процесса (`~` для home);
- **модель** — из последних строк session-jsonl (`fable-5`, `opus-4-8`, …);
- **uptime** — возраст процесса из `ps`.

## Как собираются данные (поллер раз в 5с)

| Сигнал | Источник |
|---|---|
| Список сессий | `ps -axo pid,pcpu,etime,args` → процессы, чей argv0 = `claude` |
| Проект | `lsof -p PID -a -d cwd` → последний сегмент пути |
| Модель | свежайший `*.jsonl` в `~/.claude/projects/<encoded-cwd>/`, последнее вхождение `"model":"…"` в хвосте файла |
| «Думает» | CPU процесса > 5% **или** jsonl рос за последние 10 секунд |

Каждый источник обёрнут в try/catch — если `lsof` или jsonl недоступны, строка показывается с тем, что есть, сервер не падает.

## Эндпоинты

- `GET /` — виджет (HTML)
- `GET /data` — текущий снапшот (JSON)
- `GET /events` — SSE-поток снапшотов (пуш на каждый опрос)
